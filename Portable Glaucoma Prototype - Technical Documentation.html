PORTABLE GLAUCOMA DETECTION PROTOTYPE

Computer Vision-Based CDR Analysis System

Technical Block Diagram & System Architecture Documentation

1. PROJECT OVERVIEW

The Portable Glaucoma Detection Prototype is a cost-effective, non-invasive diagnostic system designed for early-stage glaucoma screening. This system leverages computer vision techniques and machine learning to analyze the Cup-to-Disc Ratio (CDR) from fundus images captured via a macro-enabled USB camera. The processed results are displayed on an external LCD screen, making it suitable for deployment in resource-constrained healthcare settings, rural clinics, and mobile health camps.

Key Innovation: Unlike traditional fundoscopy equipment costing thousands of dollars, this prototype uses readily available components (webcam, laptop, Arduino, LCD) to provide preliminary glaucoma risk assessment at a fraction of the cost, with potential for real-time screening during mass health drives.

3. COMPONENT DESCRIPTIONS

3.1 Macro Camera / USB Webcam (Image Acquisition)

Function: Captures high-resolution close-up images of the patient's optic disc and retinal structures.

  • Hardware: Standard USB webcam with clip-on macro lens (10x-60x magnification) or varifocal webcam with macro capability
  • Resolution: Minimum 720p (1280×720), recommended 1080p for better CDR accuracy
  • Interface: USB 2.0/3.0, UVC (USB Video Class) compliant for plug-and-play operation
  • Output Format: MJPEG or YUV color space, accessible via OpenCV VideoCapture API
  • Illumination: External LED ring light (optional) for consistent fundus imaging under varying ambient conditions

3.2 Laptop / PC (Central Processing Unit)

Function: Core computational unit executing image processing pipeline and CDR analysis algorithms.

  • Software Stack: Python 3.x with OpenCV, NumPy, scikit-image, and optional TensorFlow/PyTorch for ML models
  • Processing Pipeline:
    1. Frame capture from webcam (cv2.VideoCapture)
    2. Preprocessing: Grayscale conversion, histogram equalization, noise reduction
    3. Optic disc segmentation using circular Hough transform or U-Net
    4. Cup boundary detection via adaptive thresholding
    5. CDR calculation: CDR = Cup_Diameter / Disc_Diameter
    6. Classification: NORMAL (CDR<0.3), SUSPECT (0.3-0.6), URGENT (>0.6)
  • Communication: Sends classification result as ASCII string via PySerial to Arduino
  • Min. Specs: Intel i3 / AMD Ryzen 3, 4GB RAM, USB ports, Windows/Linux/macOS

3.3 Arduino Uno / Nano (Microcontroller)

Function: Receives diagnostic results from laptop and drives the LCD display module.

  • Microcontroller: ATmega328P (16 MHz, 5V logic)
  • Serial Interface: Hardware UART on pins D0 (RX) and D1 (TX), configured at 9600 or 115200 baud
  • I2C Interface: Pins A4 (SDA) and A5 (SCL) for LCD communication using I2C backpack
  • Firmware Logic:
    1. Listen on serial port for incoming result strings
    2. Parse received data (e.g., "NORMAL", "SUSPECT", "URGENT")
    3. Update LCD display with formatted text
    4. Optional: Drive LED indicators or buzzer for audio-visual alerts
  • Libraries Used: LiquidCrystal_I2C.h or LiquidCrystal.h (depending on LCD interface)

3.4 LCD1602 Display (Result Visualization)

Function: Provides immediate, human-readable output of glaucoma screening results.

  • Display Type: 16×2 character LCD with HD44780 controller (blue/green backlight)
  • Interface Options:
    • I2C Mode (Recommended): Uses PCF8574 I2C backpack, requires only 4 connections (VCC, GND, SDA, SCL)
    • Parallel Mode: 6-pin mode (RS, EN, D4-D7) for direct Arduino connection
  • Typical Output Format: 
    Line 1: Status: NORMAL
Line 2: CDR: 0.25
  • Power Requirements: 5V DC, ~2mA (backlight off), ~20mA (backlight on)

4. DATA FLOW & PROTOCOL

End-to-End Data Pipeline

  1. Image Capture: Webcam streams video frames to laptop via USB (30-60 fps)
  2. Frame Extraction: Python script captures single frame using cv2.VideoCapture.read()
  3. Preprocessing: Contrast enhancement, bilateral filtering, ROI extraction
  4. Segmentation: Detect optic disc (largest circular feature) and cup (darker central region)
  5. CDR Calculation: Measure diameters using contour detection, compute ratio
  6. Classification: Apply threshold-based decision tree or trained ML model
  7. Serial Transmission: Send 8-byte ASCII string (e.g., "NORMAL\n") via PySerial
  8. Arduino Processing: Parse string, update LCD with formatted output
  9. Display Update: LCD shows result within 100ms of receiving data
